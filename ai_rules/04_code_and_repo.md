# 04 Code and Repository

## 当前初始化阶段

当前人类要求聚焦于 refine prompts，并把详细规则写入 `ai_rules/`。不要在没有明确要求时创建代码结构、模型 skeleton、训练入口或测试框架。

## 代码修改原则

当未来进入代码阶段时：

- 先理解现有代码和规则，再修改。
- 优先小步、可验证、可回滚的改动。
- 不做与任务无关的重构。
- 不隐藏错误，不降低验证强度。
- Python 代码应清晰、模块化，有必要的类型提示。
- 复杂逻辑用简洁中文注释解释原因，而不是复述代码。
- 配置优先于硬编码。

## 文件保护

禁止修改：

- `ai_notes/archive/` 下内容；
- `qc_*.md`；
- 人类明确标记为只读的文件；
- 未经确认的大型数据、模型权重和实验输出。

如必须修改保护内容，先提出原因并等待批准。

## Git 规则

如果仓库已初始化：

- 开始前检查 `git status --short --branch`。
- 不要覆盖他人未提交改动。
- 阶段性成果应提交，提交前说明修改范围和验证情况。
- 禁止工作完毕后不整理工作区。
- 不在不清楚分支策略时创建/切换/合并重要分支。

如果仓库尚未初始化：

- 不要擅自 `git init`，除非人类明确要求。
